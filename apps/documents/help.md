# Documents

Long-form markdown pages — notes, plans, research, checklists, specs — that you
and Skipper write, search, and grow over time.

## Overview

A document is a living markdown page (not a file attachment): always editable,
always full-text searchable, and able to link to other items (a project, a
recipe, etc.). Documents are where Skipper writes things up for you — research
findings, plans, summaries — and where several other apps store their long-form
text (research, print, brainstorming).

## Screens

- **Document list.** Browse and filter by tag; search across all docs by content.
- **Editor.** A markdown editor for reading/writing a doc; Skipper can draft and
  revise inline.
- Documents can be **linked** to a related item, so a project's docs show together.

## Example workflows

**Create a document**
- *In the app:* new document → write markdown, add tags.
- *Through chat:* "write up a packing checklist as a document", "create a research
  doc on solar panels tagged research,solar".

**Find / read one**
- *In the app:* search the list by keyword or tag.
- *Through chat:* "open my vacation plan", "search docs for solar panel costs".

**Grow a document**
- *Through chat:* "add a section on battery options to the solar doc" (appends),
  or "summarize this conversation into a new doc".

## Tips

- Tags (comma-separated) and full-text search make docs easy to find later.
- This is where Skipper co-authors specs and plans with you.
- For binary files (PDFs, images) use attachments on the relevant item instead.

## Your data

Documents are **saved in the database, full-text searchable, and pulled into
Skipper's memory** — so you can ask "what did we write about solar?" and Skipper
finds it. They stay within your household.
